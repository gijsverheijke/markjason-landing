AGENTS.md Editor

AGENTS.md is becoming the standard way to tell AI coding agents how to work with your project. Here's how markjason makes editing these files fast and effective.

What is AGENTS.md?

AGENTS.md is a markdown file that defines how AI coding assistants should behave in your project. It's like a README, but for robots.

Different tools have their own conventions:

  • AGENTS.md — Generic AI agent instructions (growing standard)
  • CLAUDE.md — Claude-specific project instructions
  • .cursorrules — Cursor AI rules (markdown format)
  • .github/copilot-instructions.md — GitHub Copilot guidance

The content is similar: project context, coding conventions, file structure, and instructions the AI should follow.

Why AGENTS.md Matters

AI coding assistants are only as good as their context. A well-written AGENTS.md can:

  • Set coding standards — naming conventions, patterns, style
  • Explain architecture — how the codebase is organized
  • Define boundaries — what the agent should and shouldn't touch
  • Provide context — domain knowledge, business logic
  • Speed up onboarding — new agents (and humans) get up to speed fast

This is prompt engineering at the project level.

Example AGENTS.md Structure

Here's a typical structure:

# AGENTS.md

## Project Overview
Brief description of what this project does.

## Tech Stack
- Language: TypeScript
- Framework: Next.js 14
- Database: PostgreSQL with Drizzle ORM

## Architecture
- `/app` — Next.js App Router pages
- `/lib` — Shared utilities and configs
- `/components` — React components

## Coding Conventions
- Use functional components with hooks
- Prefer named exports over default
- Write tests for new features

## Things to Avoid
- Don't modify migrations directly
- Don't expose API keys in client code
- Don't use `any` type

## Commands
- `pnpm dev` — Start development server
- `pnpm test` — Run tests
- `pnpm db:push` — Push schema changes

Best Practices for AGENTS.md

Keep It Concise

Every token in AGENTS.md is a token not available for code context. Be brief but complete.

Bad:

Please always remember to use TypeScript for all code in this project. TypeScript is our language of choice because it provides type safety and better developer experience...

Good:

Use TypeScript. No exceptions.

Be Specific About Patterns

Vague instructions get vague results.

Bad:

Write clean code.

Good:

- Functions under 30 lines
- No nested ternaries
- Extract magic numbers to constants

Include Real Examples

Show, don't just tell.

## Component Pattern

Example:
```tsx
export function UserCard({ user }: { user: User }) {
  return (
    <div className="card">
      <h3>{user.name}</h3>
    </div>
  );
}
```

Update Regularly

Your AGENTS.md should evolve with your project. After finding a pattern that works (or doesn't), update the instructions.

AGENTS.md for Different Tools

Claude Code

Claude Code reads CLAUDE.md or AGENTS.md in your project root. It also respects .claude directory for project settings.

Cursor

Cursor uses .cursorrules in your project root. Same markdown format, different filename.

GitHub Copilot

Copilot Workspace reads from .github/copilot-instructions.md.

Generic Setup

If your tool doesn't have a specific convention, use AGENTS.md in the project root. It's becoming the de facto standard.
